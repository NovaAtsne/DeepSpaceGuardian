# 贡献指南

感谢您考虑为本项目，进行贡献，以下是贡献指南：

##### 1. Fork目标仓库

- 打开DeepSpaceGuardian的 [GitHub](https://github.com/NovaAtsne/DeepSpaceGuardian) 页面，点击右上角的 **Fork** 按钮，将项目复制到你的账户下。
- 这样你就拥有了该项目的副本，可以自由修改而不影响原始仓库。

##### 2. 克隆到本地

使用以下命令将 Fork 的仓库克隆到本地（前提是你要先安装Git）：

```bash
git clone git@github.com:你的用户名/项目名.git
cd 项目名
```

###### 3. 创建功能分支

在进行任何修改前，创建一个新的分支以保持主分支的干净：

```bash
git checkout -b feature/你的功能描述
```

##### 4. 修改代码并提交

在本地编辑代码，完成后将更改添加到暂存区并提交：

```bash
git add
git commit -m "fix: 描述你的更改"
```

##### 5. 推送到远程仓库

将功能分支推送到你 Fork 的远程仓库：

```bash
git push origin feature/你的功能描述
```

##### 6. 创建 Pull Request

打开你的 Fork 仓库，点击 **Compare & pull request** 按钮。填写标题和描述，说明更改内容及其目的。提交后等待项目维护者的审核。

##### 7. 与维护者协作

根据维护者的反馈修改代码，并推送更新：

```bash
git add
git commit -m "fix: 根据反馈调整代码"
git push origin feature/你的功能描述
```

最佳实践

**从小问题入手**：例如修复文档或简单 Bug，熟悉项目流程。

**遵循贡献指南**：阅读项目的 **CONTRIBUTING.md** 文件，了解编码规范和提交要求。

**保持分支同步**：定期拉取上游更新以减少冲突：

```bash
git fetch upstream
git merge upstream/main
```
